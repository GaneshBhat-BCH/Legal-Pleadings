# Docker Deployment Guide for Legal Pleadings API

This guide covers everything you need to know to permanently host the Legal Pleadings backend using Docker on your Windows Virtual Machine.

---

## 1. Install Docker Desktop (Command Line)
If your Virtual Machine does not have `winget` installed, you can silently download and install the official Docker executable directly from PowerShell.

1. Open **PowerShell** as an Administrator.
2. Run the following command block exactly as written to download the `.exe` and install it silently:
   ```powershell
   Invoke-WebRequest -Uri "https://desktop.docker.com/win/main/amd64/Docker%20Desktop%20Installer.exe" -OutFile "C:\DockerInstaller.exe"
   Start-Process "C:\DockerInstaller.exe" -ArgumentList "install", "--quiet" -Wait
   Remove-Item "C:\DockerInstaller.exe"
   ```
3. Once the installation completes, **restart your Virtual Machine**.
4. After restarting, open the **Docker Desktop** app from your Start Menu once to accept the terms and let it start its background engine.
   - *Important:* Go to Docker Desktop Settings (the gear icon) > General > Check the box for **"Start Docker Desktop when you log in"**.

---

## 2. Start the Server (Daemonize)
To boot up the FastAPI server so that it runs invisibly forever in the background:

1. Open PowerShell and navigate to your project folder:
   ```powershell
   cd "C:\Users\srvidhCH273818\Documents\Legal Pleadings"
   ```
2. Build and start the container in detached mode (`-d`):
   ```powershell
   docker-compose up -d --build
   ```
*Your server is now running on port 8000. It will automatically restart itself if it crashes or if the VM reboots.*

---

## 3. Monitor the Server Logs
If you want to watch the real-time API logs (e.g., to watch extractions processing):

```powershell
docker-compose logs -f
```
*(Press `Ctrl + C` to stop watching the logs. This does NOT stop the server, it just exits the log screen).*

---

## 4. Stop the Server
If you need to completely shut down the API (for example, to free up memory):

```powershell
docker-compose down
```

---

## 5. Restart / Update the Server
If you change any Python code, update the `.env` file, or pull new code from Git, you must restart the container for the changes to take effect:

1. Rebuild and restart the container:
   ```powershell
   docker-compose up -d --build
   ```
*(You do not need to run `down` first; running `up --build` will automatically replace the old version with your new code).*

---

## Important File Locations
Because the API is running inside an isolated Docker container, we have mapped key folders so your host VM can still access the generated files:
* **Logs:** The `activity_log.csv` and `ai_error_log.csv` will still appear normally in `backend/logs/`.
* **Word Documents:** The `.docx` Position Statements generated by the AI will be deposited into `C:\Users\srvidhCH273818\Documents\Legal Pleadings\Downloads`.
